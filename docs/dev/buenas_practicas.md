# Buenas Prácticas de Desarrollo

## Flujo de trabajo
- PRs pequeños, orientados a un ticket.
- Commits descriptivos y atómicos.
- Validar localmente antes de pedir revisión.

## Calidad
- Mantener cambios acotados al alcance solicitado.
- Preservar compatibilidad de navegación y estado.
- Evitar duplicar lógica; extraer helpers cuando aporte claridad.

## UI/UX
- Priorizar legibilidad (contraste, jerarquía visual, textos cortos).
- Mantener consistencia de labels y acciones.
- Añadir feedback visible en acciones críticas (confirmación/estado/error).

## Datos y lore
- JSON válido, claves estables, fallback seguro.
- Textos en español revisados por tono y longitud.
