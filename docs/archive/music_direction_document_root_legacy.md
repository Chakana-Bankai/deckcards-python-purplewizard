# CHAKANA Music Direction Document

## Goal
Guide procedural music generation toward distinct, emotional, and non-repetitive context identity.

## Core Structure by Context
Required contexts:
- menu
- map
- combat
- boss

Each track must include:
1. Melody
2. Harmony
3. Rhythm

## Context Direction

### Menu
- Mood: mystical, calm, ceremonial.
- Melody: long phrases, airy motif.
- Harmony: suspended/ritual chords.
- Rhythm: soft pulse, low density.

### Map
- Mood: exploration + sacred tension.
- Melody: evolving motif with biome color.
- Harmony: modal movement, gradual transitions.
- Rhythm: meditative but directional.

### Combat
- Mood: tactical urgency.
- Melody: short assertive motifs, clear contour.
- Harmony: tighter progression, darker accents.
- Rhythm: energetic but readable, no chaotic noise.

### Boss
- Mood: techno-ritual escalation.
- Melody: ceremonial lead with dramatic peaks.
- Harmony: heavier minor dissonance + release.
- Rhythm: layered drive, strong identity.

## Anti-Repetition Rules
- Rotate rhythmic templates per context.
- Avoid persistent high-pitched whistle tone.
- Use motif families tied to biome/faction.
- Detect near-identical output signatures and regenerate.

## Mix and Runtime Rules
- Keep headroom and avoid clipping.
- Favor longer phrase arcs over short hypnotic loops.
- Preserve cache determinism.
- Curated assets override generated when present.
