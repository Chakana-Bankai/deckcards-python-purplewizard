[tracks_too_slow_or_heavy]
combat_boss_a -> low BPM identity is valid, but still leans too close to ceremonial dark sustain as default boss language.
defeat_a -> current defeat lane remains very low-energy and old-style somber.
map_ukhu_a -> ambience-heavy and closer to drone travel than alive exploration.
shop_a -> intimate but still too static compared to the new luminous ritual target.

[tracks_too_monochromatic_or_placeholder]
menu_a -> better than older versions, but still long-form and not yet the preferred colorful canonical main_menu asset.
combat_a -> carries movement, but remains too close to the old unified combat color instead of stronger contextual contrast.
reward/victory overlap -> reward identity is still underdeveloped in runtime because reveal moments still collapse into victory-like behavior in asset coverage.
legacy sfx set -> button_click/card_play/card_invalid are functional but still read as utility bleeps rather than a cohesive ritual UI language.

[tracks_using_overly_dark_or_gothic_defaults]
archon_void_drones and some older map/boss assets still pull the whole sound world toward dark ritual drone language.
The old system still biases toward low choir + drone + sustained dark pad as a safe fallback.
Credits and codex have corrected routing in code, but not yet corrected runtime asset replacement.

[missing_melodic_identity]
main_menu lacks a clearly promoted canonical melodic runtime theme.
map_exploration lacks a dedicated alive exploratory identity in runtime.
reward, codex and credits lack active distinct assets despite now having canonical specs.
rare_reveal and legendary_reveal exist in MVP v2 but are not yet the live default reveal assets.

[missing_contextual_differentiation]
reward and reveal moments are still not fully separated from victory/shop era behavior in the active asset pool.
map contexts are split, but aliases still collapse map_exploration to map_kay.
combat_normal/combat_elite/combat_boss are not yet rebuilt under one corrected colorful runtime family.

2. long BGM assets still keep redundant tails after the useful loop closure.
3. multiple contexts have corrected specs but no promoted runtime replacements yet.
4. SFX coverage is incomplete, so many actions still share generic tones.
5. reveal/reward/codex/credits identity exists in data but not yet in active asset assignment.
