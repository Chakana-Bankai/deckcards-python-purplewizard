summary
- Integrated soundfile into the audio pipeline for safer WAV write/read handling.
- Integrated pydantic audio schemas for asset specs and analysis payloads.
- Integrated rich logging into the runtime audio engine for clearer generation/playback reporting.
- Integrated guarded librosa availability into the analysis layer while keeping runtime-safe soundfile/numpy analysis active.
- Added manifest backfill for audio analysis metadata on cached/generated assets.

files_modified
- game/audio/audio_stack_tools.py
- game/audio/audio_engine.py

integration_details
- soundfile:
  - game/audio/audio_stack_tools.py::write_wav_soundfile now writes PCM_16 wav assets.
  - game/audio/audio_stack_tools.py::analyze_audio_file reads wav metadata and sample windows safely.
  - game/audio/audio_engine.py::_write_wave now delegates to soundfile-backed writing.
- librosa:
  - game/audio/audio_stack_tools.py imports librosa in guarded mode and records librosa availability in analysis payloads.
  - Runtime analysis uses a guarded fallback mode because deeper librosa onset/tempo routines stall in this local Python 3.14 environment.
- pydantic:
  - game/audio/audio_stack_tools.py::AudioAssetSpec validates audio write specs.
  - game/audio/audio_stack_tools.py::AudioAnalysisReport validates exported analysis payloads.
- rich:
  - game/audio/audio_engine.py now uses rich console logging for generation, playback, ambient and error messages.
